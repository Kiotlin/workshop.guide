# Workshop.guide 是什么

**Workshop.guide** 是一个个人维护的工作车间，记录了通过互联网已知的、或实际生产过程中使用过的各种软件，以及各种有价值的信息渠道。

## 为什么需要记录

这个问题要分为多种场景来看：
- Web application: web程序可能是当今时代最方便使用的程序类别之一，我们只需要输入网址便可以快速访问与使用，但相对应的管理问题也十分显著。对于一个没有维护书签标题习惯的浏览器使用者，长串的书签栏以及下拉菜单往往不能帮助我们快速找到我们想要的web服务。其次，某些web服务使用的频率是非常离散的。可能你今天精心挑选并使用的图床服务，再次使用已经是3个月以后了，随着你按下浏览器的关闭按钮以及时光流逝，当你再次出现图床需求时，你需要重新花费时间去搜索引擎寻找事宜你的产品，这是一个非常折磨并且没有效率的行为，因此及时记录下使用过的高质量产品是非常必要的。
- Desktop application: 除了跟web应用相同的再搜索成本，桌面应用还有一个特点就是会随着重装系统而从你的计算机中被移除，重装系统虽然不是一个非常高频的行为，但为了重新找回与你在上一个版本系统中相当的用户体验，往往需要伴随着经常使用应用的再安装，维护一个常用软件列表是有必要的。桌面应用同样存在使用频率离散的情况，因此做好记录是一个很好的好习惯。
- Creative page: 单纯得享受web世界给我们带来的便捷，灵感与惊叹。这些内容可能本身或永远都与你毫不相干，但作为旁观者第一次触及它们所给你带来的惊艳是让你难以忘怀的，就宛如走进一座精美的博物馆，如果不将这些记忆及时地保存下来，它们将可能永远被遗忘在你的时间卷轴之中。

## 分类的原则
1. 分类不能太细。
2. 常用的工具做上标记，尽量与不常用的工具区分开。
3. 相似功能的应用尽量放在一起，便于筛选。
4. 对软件功能做好详细的注释，一眼读完便能知其应用场景即可。

## 这个网站是怎么做的

**Workshop.guide** 使用了[Nextra](https://nextra.vercel.app/)来做静态网站生成。

**Nextra** 是一个基于[Next.js](https://nextjs.org/)和[MDX](https://mdxjs.com/)的"no-code"型网站生成器。
